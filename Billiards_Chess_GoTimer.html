소개

Multi Game Clock 앱(이하 “앱”)은 당구·체스·바둑 타이머 기능을 제공합니다. 현재 버전은 광고 및 제3자 SDK가 없으며, 단말 내에서 타이머 기능만 수행합니다.

우리는 사용자의 개인 데이터를 수집·저장·처리·공유하지 않습니다.

  • 계정, 이메일, 이름 등의 식별자 미수집
  • 점수/설정 등 앱 내 데이터는 로컬에서만 사용되며 외부로 전송하지 않습니다.

권한

앱은 동작에 필요한 최소 권한만 사용합니다. 현재 버전 기준:

  • 네트워크 권한 없음(광고/분석 미사용)
  • (선택적으로 도입 가능) WAKE_LOCK: 경기 중 화면 꺼짐 방지 옵션 제공 시 사용
  • (선택적으로 도입 가능) 진동/사운드 사용 시 관련 권한 또는 기기 기능 접근

디바이스 기능

  • 타이머 정확도를 위해 기기 시간을 사용합니다.
  • 선택 기능(도입 시): 화면 꺼짐 방지, 진동/사운드 알림

제3자 서비스

Introduction

The Multi Game Clock app provides timers for Billiards, Chess, and Go (Baduk). The current version has no ads and no third-party SDKs, and runs timers locally on the device.

Data We Collect

We do not collect, store, process, or share any personal data.

  • No collection of identifiers (accounts, emails, names, etc.).
  • Scores and settings are used locally and are not transmitted.
  • No analytics/logging to remote servers (no network dependency).

Permissions

The App uses only minimal permissions. In the current release:

  • No network permissions (no ads/analytics).
  • (Optionally, if enabled) WAKE_LOCK to keep the screen on during matches.
  • (Optionally, if enabled) Vibration/Sound features may access device capabilities.

Device Features

  • The timer relies on the device clock for accuracy.
  • Optional features (if provided): keep screen on, vibration/sound cues.

Third-Party Services

The current version does not include any third-party SDKs (ads, analytics, crash reporting, authentication, etc.).

Advertising / Monetization

No ads are shown in this release. If we introduce ads or analytics in the future, we will update this Policy and include links to the relevant SDK policies.

Children’s Privacy

The App is not directed to children, and we do not knowingly collect personal data from children.

Security

  • The App does not transmit data to external servers.
  • We recommend standard device security practices (screen lock, OS updates, etc.).

Changes to This Policy

We may update this Policy from time to time. We will update the “Last updated” date above and, for material changes, provide notice in-app or via store release notes.
